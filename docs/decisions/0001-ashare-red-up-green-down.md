# 0001. A 股配色:红涨绿跌

- 状态:已采纳
- 日期:2026-07-08

## 背景

本站讲 A 股,配色是内容语义的一部分——多处正文直接写「红色代表上涨」。而国际惯例(以及 klinecharts 默认主题)是绿涨红跌,正好相反。若不统一,图和文字会打架、误导新手。

## 决策

全站涨跌语义色一律 **红 = 涨/阳、绿 = 跌/阴**,用 CSS 变量 `--stock-up`(红)、`--stock-down`(绿)承载(各带 `-soft` 浅底),亮暗两套都遵循。

## 理由

- 面向 A 股读者,红涨绿跌是本土直觉,与正文表述一致。
- 收敛成两个语义变量,组件只引用变量而非具体颜色,便于全局一致与暗色适配。
- 参考的 klinecharts 用绿涨红跌,但那是「参考渲染形态」,颜色按本站约定重映射即可。

## 后果

- 从 klinecharts / 国际图表迁移形态时,必须把颜色重映射到本站变量,不能照搬。
- 图表组件禁止写死涨跌 hex(配套红线 + `docs/modules/diagrams/`),否则暗色与一致性会破。
