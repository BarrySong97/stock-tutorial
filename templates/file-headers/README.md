# 文件头规范(AI 文件头)

每个源文件**顶部**放一段轻量注释,专门给 AI 看,让它不读完整文件也能知道这文件的角色。

## 四个字段(固定)
- `@purpose` — 这个文件是干嘛的(一句话)。**check-docs 用它做识别标记,必须有。**
- `@role` — 在系统/模块里扮演的角色;被谁调用、调用谁。
- `@deps` — 关键依赖(模块、库、外部服务)。
- `@gotcha` — 坑、约束、不变量;相关模块文档链接(如 `docs/modules/payments/README.md`)。

## 原则
- **轻量**:只写职责级信息,**不写函数签名/参数**——那些看代码即可,且最容易过期。
- 用各语言自己的注释语法(见本目录下 `*.txt`),四个字段保持一致。
- 文件职责变了就更新文件头;`node scripts/check-docs.mjs` 会揪出缺头的文件。
